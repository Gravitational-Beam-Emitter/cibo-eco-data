# Product

## Register

product

## Users

炒股个人投资者。A股收盘后（15:00-18:00）快速复盘当日涨停板：
- 扫一眼市场主线，知道今天在炒什么
- 按行业/连板筛选涨停股，找机会
- 跟踪关注个股的涨停历史

用户不是机构分析师，不需要 Bloomberg 级终端。需要的是清晰、快速、友好地完成复盘。

## Product Purpose

将 niushia.com 的核心价值（涨停原因标签 + 市场主线归纳）做成一个对外的 Web 产品。
后端自动拉取数据、LLM 打标，前端提供流畅的复盘浏览体验。
成功 = 用户每天收盘后打开这个页面，5 分钟内完成复盘。

## Brand Personality

现代、清晰、友好。

- 现代：不是传统的财经终端，不是满屏表格
- 清晰：信息层次分明，一眼看到重点
- 友好：不吓人，不冰冷，有温度但保持专业

## Anti-references

- 同花顺/东方财富风：太拥挤、广告轰炸、找不到重点
- Bloomberg 终端风：太冷、太密、高学习门槛
- AI 塑料感：Inter 字体 + 紫蓝渐变 + 纯白卡片堆叠，一眼假
- 老派交易软件：厚重边框、表格堆砌、无信息层级

## Design Principles

1. **一眼看清主线** — 打开页面首先看到的是市场主线，而不是数据表格
2. **层层递进** — 概览 → 主线 → 股票列表 → 个股详情，按用户的浏览节奏来
3. **信息密度适度** — 不挤不空。比 Bloomberg 松，比 landing page 密
4. **标签即答案** — LLM 生成的涨停原因标签是核心价值，要突出展示
5. **熟悉但不俗** — 用户认得出来是财经工具，但不会是老派交易软件的样子

## Visual Direction

Robinhood 极简风 + 中国市场规矩。

- **底色**：纯中性暗黑（`oklch(0.105 0 0)`），无色偏
- **Primary**：克制暖红（`oklch(0.54 0.165 22)`），不刺眼不浮躁
- **Up/Down**：红涨绿跌，遵守中国市场习惯
- **无边框**：表面通过微妙背景色差区分，不依赖 border
- **数字优先**：大字号涨跌幅，小标签辅助
- 灵感：Robinhood 的极简克制 + 中国股市的红涨语境

## Accessibility & Inclusion

基础可访问性：保证颜色对比度达标，支持键盘导航，减少动效或提供关闭选项。
